## 1. 范围与目标

本规范定义 Cavvy 包管理器及兼容实现在处理网络来源库时的安全策略与交互规范。规范聚焦于**来源识别、风险分级与用户告知**，不涉及包管理器的具体实现细节、命令行接口或内部架构设计。

目标包括：
- 统一不同来源类型的安全处理策略；
- 确保用户在安装非官方认证内容前获得明确、不可忽略的风险提示；
- 为客户端实现提供行为层面的合规标准，而非代码层面的实现约束。

## 3. 术语与定义

| 术语 | 定义 |
|------|------|
| **纯包名（Bare Package Name）** | 用户仅提供包标识符，未附带任何 URL、源地址或仓库信息。 |
| **直接 URL（Direct URL）** | 用户显式提供 HTTP(S) 或 SSH 格式的完整仓库地址，用于定位并安装软件包。 |
| **自定义源（Custom Source）** | 用户配置的第三方源服务器地址，配合包名进行安装。 |
| **官方安全源（Official Secure Source）** | 由 Ethernos 官方认证并列入 `package.cavvy.ethernos.net` 索引的来源。 |
| **未验证来源（Unverified Source）** | 未通过 Ethernos 官方认证，或无法完成 ESSO-10430 证书验证链的来源。 |
| **阻塞警告（Blocking Warning）** | 一种交互式安全提示，要求用户显式确认后方可继续操作，默认阻塞流程推进。 |
| **静默警告（Silent Warning）** | 仅输出警告文本至标准输出或日志，不阻塞操作流程。 |

---

## 4. 来源分类体系

网络来源库按用户输入形式分为三类：

| 类别 | 输入形式示例 | 来源确定性 |
|------|-------------|-----------|
| **A 类：纯包名** | `cavvy-std` | 必须通过官方安全源索引解析 |
| **B 类：直接 URL** | `https://github.com/user/repo` 或 `git@github.com:user/repo` | 由 URL 直接定位，绕过官方索引 |
| **C 类：自定义源 + 包名** | `--source https://custom.registry.net pkg-name` | 由第三方源服务器解析 |

---

## 5. 处理策略总览

| 来源类别 | 安全假设 | 默认行为 | 警告级别 | 用户可关闭阻塞 |
|----------|----------|----------|----------|----------------|
| A 类：纯包名 | 官方认证安全库 | 直接查询官方索引，验证证书后安装 | 无 | 不适用 |
| B 类：直接 URL | 未验证来源 | 不阻止安装，触发阻塞警告 | 阻塞警告 | 是 |
| C 类：自定义源 + 包名 | 未验证来源 | 不阻止安装，触发阻塞警告 | 阻塞警告 | 是 |

---

## 6. 纯包名来源处理（A 类）

### 6.1 解析流程

1. 客户端将包名提交至官方安全源索引服务器（`package.cavvy.ethernos.net`）；
2. 若索引中存在该包名，提取其包指纹、最新版本及证书地址；
3. 按 ESSO-10430 规范完成证书链验证与完整性校验；
4. 验证通过后执行安装。

### 6.2 未命中处理

若官方索引中不存在该包名：
- 视为安装失败；
- 向用户输出提示信息，说明该包未通过官方认证；
- 不得自动回退至其他源进行搜索或安装。

### 6.3 安全保证

A 类来源在正确实现下，保证用户安装的包满足：
- 来源身份经过 Ethernos 官方认证；
- 内容完整性经过 SHA-256 校验；
- 发布者身份经过 Ed25519 双重签名验证。

## 7. HTTP/SSH URL 来源处理（B 类）

### 7.1 行为原则

- **不阻止安装**：客户端不得因来源未经验证而直接拒绝安装；
- **强制阻塞警告**：必须在安装前向用户展示阻塞警告，要求显式确认；
- **允许关闭阻塞**：用户可通过配置关闭阻塞警告，但关闭后必须输出静默警告文本。

### 7.2 阻塞警告内容要求

阻塞警告必须至少包含以下信息：

```
[SECURITY WARNING]
You are about to install a package from an unverified source.

Source URL: {URL}
This package has NOT been certified by the Ethernos official secure source system.
The content has not been verified for integrity or authenticity.

Risks include but are not limited to:
  - Malicious code execution
  - Data corruption or loss
  - Supply chain compromise

To proceed, explicitly confirm your understanding of these risks.
```

- 警告文本必须以 `[SECURITY WARNING]` 或等效标识开头；
- 必须明确告知用户该来源**未经过 Ethernos 官方认证**；
- 必须列出潜在风险摘要；
- 必须要求用户显式输入确认（如 `yes` / `y`）或点击确认，而非默认回车继续。

### 7.3 关闭阻塞后的静默警告

若用户通过配置关闭阻塞警告，客户端仍必须在标准输出（或等效日志通道）打印以下文本：

```
[SECURITY NOTICE] Installing from unverified source: {URL}. 
Integrity and authenticity verification skipped. 
This installation is not covered by Ethernos secure source guarantees.
```

- 该文本不得以日志级别过滤（如 `debug` / `trace`）而被静默丢弃；
- 必须输出至至少 `warning` 级别或标准输出流。

## 8. 自定义源服务器来源处理（C 类）

### 8.1 行为原则

与 B 类来源一致：
- **不阻止安装**；
- **强制阻塞警告**；
- **允许关闭阻塞**，但关闭后必须输出静默警告文本。

### 8.2 阻塞警告内容要求

```
[SECURITY WARNING]
You are about to install a package from a custom source server.

Source Server: {SOURCE_SERVER_URL}
Package Name: {PACKAGE_NAME}
This source server is NOT part of the Ethernos official secure source network.
Packages from this server have not been certified by Ethernos.

Risks include but are not limited to:
  - Untrusted package metadata
  - Potential man-in-the-middle attacks
  - Absence of official integrity verification

To proceed, explicitly confirm your understanding of these risks.
```

### 8.3 关闭阻塞后的静默警告

```
[SECURITY NOTICE] Installing from custom source server: {SOURCE_SERVER_URL}. 
Package: {PACKAGE_NAME}. 
Official secure source verification not applicable. 
This installation is not covered by Ethernos secure source guarantees.
```

### 8.4 自定义源的证书验证（可选增强）

若自定义源服务器实现了 ESSO-10430 兼容的证书接口：
- 客户端**可**尝试验证其证书链；
- 验证成功**不改变**其未验证来源的分类（因非官方根密钥签名）；
- 验证失败**不得**阻止安装，但必须在警告中附加证书验证失败信息。

## 9. 阻塞警告机制

### 9.1 机制定义

阻塞警告是一种**问答式（Interactive）**安全提示，具备以下特征：

- **流程阻塞**：警告展示期间，安装流程暂停，等待用户输入；
- **默认安全**：用户直接回车或输入无效内容时，默认行为必须为**取消安装**；
- **显式确认**：用户必须输入明确的确认词（如 `yes`、`confirm`）或选择确认选项；
- **单次会话**：用户在一次会话中对同一来源的确认，可缓存至会话结束，避免重复提示；
- **跨会话失效**：新会话重新启动时，缓存的确认决策失效，必须重新提示。

### 9.2 配置关闭机制

用户可通过以下方式关闭阻塞警告（具体配置键由客户端实现定义）：
- 全局配置文件中的安全级别降级选项；
- 环境变量（如 `ESSO_UNVERIFIED_SOURCE_NO_BLOCK=1`）；
- 命令行标志（如 `--no-source-block`）。

**关闭后的强制要求**：
- 无论通过何种方式关闭，客户端必须输出第 7.3 节与第 8.3 节规定的静默警告文本；
- 关闭配置本身必须被记录至审计日志（参见第 10 节）。

### 9.3 非交互环境处理

在非交互环境（如 CI/CD 流水线、脚本执行）中：
- 若阻塞警告未关闭，安装必须失败并返回非零退出码；
- 失败信息必须明确说明失败原因："Unverified source installation requires explicit confirmation in interactive mode, or configuration override in non-interactive mode."

## 10. 日志与审计要求

### 10.1 强制记录事件

以下事件必须记录至客户端日志：

| 事件 | 日志内容 |
|------|----------|
| 安装未验证来源（B 类） | 时间戳、URL、用户确认方式（阻塞确认 / 关闭后静默安装） |
| 安装自定义源（C 类） | 时间戳、源服务器 URL、包名、用户确认方式 |
| 关闭阻塞警告配置 | 时间戳、配置项、配置值、生效范围（全局 / 会话） |
| 官方安全源验证失败 | 时间戳、包指纹、失败原因 |

### 10.2 日志级别

- 未验证来源安装事件：至少 `warning` 级别；
- 关闭阻塞警告配置变更：至少 `info` 级别；
- 安全验证失败：至少 `error` 级别。

---

## 11. 安全基线衔接

本规范直接继承 ESSO-10400 的安全基线：

- **默认安全**：A 类来源默认走官方安全源，B/C 类来源默认触发警告；
- **故障安全**：非交互环境下未验证来源安装默认失败；
- **透明可审计**：所有未验证来源安装及配置变更强制记录日志。

本规范与 ESSO-10410 的衔接：
- A 类来源的验证依赖 ESSO-10410 定义的 SHA-256 与 Ed25519 标准；
- B/C 类来源因绕过官方索引，无法应用完整的双重签名验证链。

## 12. 附录

### 附录 A：来源类型快速判定流程

```
用户输入
    │
    ├─ 纯包名（无协议头、无源参数）→ A 类：查询官方索引
    │
    ├─ 以 http:// / https:// / git@ 开头 → B 类：URL 直接安装
    │
    └─ 指定 --source 或等效自定义源参数 → C 类：自定义源安装
```

### 附录 B：警告文本本地化建议

警告文本的英文版本为规范强制要求。客户端实现可提供本地化翻译，但：
- 翻译版本必须完整保留原意，不得弱化风险描述；
- 英文原文必须同时输出或可通过 `--verbose` 等标志查看。
